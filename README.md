# LinkTheory
# LinkTheory: Ассоциативная Операционная Система (прототип)

Проект по реализации прикладной модели данных на основе "The Links Theory" ($L \to L^2$). 
Мы отказываемся от иерархических папок и таблиц в пользу единого графа связей.

## Зачем этот репозиторий?
Цель проекта — доказать, что любая информация (файлы, БД, код) может быть эффективно представлена в виде **дуплетов** (связей между двумя ID). 

### Ключевые преимущества:
1. **Глобальная дедупликация**: Одинаковые значения (например, категория "Электроника" в 1000 строк) хранятся как одна точка в памяти.
2. **Многомерность**: Один объект может находиться в бесконечном количестве контекстов (папок/тегов) без копирования данных.
3. **Скорость**: Поиск по ассоциациям работает быстрее, чем традиционные JOIN-запросы в SQL.

## Что мы тестируем (Milestone 1)
Мы берем массив данных (CSV на 1000+ строк) и "размалываем" его в атомарные связи. 

**Задача:** 
Показать, что поиск транзакций по категории через граф связей — это естественная операция системы, а не надстройка над файлом.

## Структура проекта
- `core.py`: Ядро системы. Реализация хранилища дуплетов в SQLite.
- `fs.py`: Слой файловой системы. Эмуляция папок и файлов через связи.
- `importer.py`: Инструмент для импорта CSV-массивов в ассоциативную среду.
- `shell.py`: (В разработке) CLI для навигации по связям.

## Как запустить тест?
1. Запустите `importer.py` для генерации данных и создания базы `links.db`.
2. Система создаст ~4000 уникальных связей для 1000 строк CSV.
3. Проверьте работоспособность, выполнив поиск по ID категории.
